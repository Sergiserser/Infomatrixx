# Объяснение программы `rescue app.py`

Этот файл объясняет, что делает программа и где в коде находятся основные части.

## Главная идея

`rescue app.py` - это прототип помощника для экстренных ситуаций. Он одновременно смотрит данные с микрофона и камеры, оценивает риск и при высокой опасности включает тревогу с обратным отсчетом.

По умолчанию программа работает в безопасном DEMO-режиме: она показывает тревогу и сохраняет доказательства, но не звонит на реальный номер сама. Это специально сделано, чтобы случайный тест не отправил ложный вызов.

## Как программа работает по шагам

1. Открывает камеру.
2. Пытается включить микрофон.
3. В реальном времени анализирует звук.
4. В реальном времени анализирует видео.
5. Анализирует жесты помощи: кулак и большой палец вверх.
6. Складывает признаки опасности в общий риск `0-100`.
7. Если риск выше порога, включает тревогу.
8. Переключается на вкладку-ленту ближайших укрытий.
9. Сохраняет кадр и JSON-описание события в папку `emergency_evidence`.
10. Показывает обратный отсчет.
11. Пользователь может нажать `X`, чтобы отменить, или `C`, чтобы подтвердить действие.
12. Если включен реальный режим, программа может отправить звонок через Twilio или выполнить локальную команду.
13. Пользователь может привязать Google-аккаунт через клавишу `L`.

## Основные блоки кода

### `RescueConfig`

Класс с настройками программы.

В нем задаются:
- номер экстренной службы;
- режим DEMO или REAL;
- номер камеры;
- частота аудио;
- порог риска;
- время обратного отсчета;
- настройки Twilio;
- команда для локального действия.
- настройки Google Maps;
- настройки Google OAuth.

Настройки можно менять через переменные окружения, например:

```powershell
$env:EMERGENCY_NUMBER="0977477926"
$env:RISK_THRESHOLD="70"
$env:EMERGENCY_COUNTDOWN_SECONDS="8"
```

Номер по умолчанию сейчас:

```text
0977477926
```

### `GoogleAccountManager`

Отвечает за регистрацию/вход через Google-аккаунт.

Как работает:
- открывает страницу входа Google в браузере;
- принимает ответ на локальный адрес `127.0.0.1`;
- получает публичный профиль пользователя;
- сохраняет только `sub`, `email`, `name`, `picture` в `google_user_profile.json`;
- OAuth-токены не сохраняет.

Для настройки нужны OAuth credentials из Google Cloud:

```powershell
$env:GOOGLE_OAUTH_CLIENT_ID="your_client_id"
$env:GOOGLE_OAUTH_CLIENT_SECRET="your_client_secret"
```

В Google Cloud лучше создавать OAuth Client типа `Desktop app`. Программа использует локальный redirect на `http://127.0.0.1:<random_port>` и PKCE.

После запуска нажми `L`, чтобы зарегистрироваться или войти через Google.

Если нужно требовать вход перед запуском мониторинга:

```powershell
$env:GOOGLE_LOGIN_REQUIRED="1"
```

### `AudioAnalyzer`

Отвечает за анализ микрофона.

Он ищет:
- резкий громкий импульс, похожий на выстрел;
- длительный громкий высокочастотный звук, похожий на крик.

Программа не "понимает" звук как человек. Она смотрит на технические признаки:
- громкость;
- пик звука;
- насколько резко звук вырос;
- сколько энергии в высоких частотах;
- насколько звук похож на короткий ударный импульс.

Когда найден опасный звук, создается `AudioEvent`.

### `GestureAnalyzer`

Отвечает за распознавание жестов помощи через MediaPipe.

Сейчас добавлены:
- `thumbs_up_help` - большой палец вверх;
- `fist_help` - кулак.

Чтобы случайный жест не запускал тревогу, его нужно удерживать примерно `1.1` секунды. Это можно изменить:

```powershell
$env:GESTURE_HOLD_SECONDS="1.5"
```

### `VideoAnalyzer`

Отвечает за анализ камеры.

Он ищет:
- внезапную вспышку;
- резкое движение в кадре.

Для этого каждый новый кадр сравнивается с предыдущим. Если картинка резко изменилась или стала намного ярче, создается `VideoEvent`.

### `RiskFusion`

Это блок принятия решения.

Он получает события от аудио и видео и увеличивает общий риск. Например:
- звук, похожий на выстрел, сильно повышает риск;
- несколько импульсов подряд повышают риск еще сильнее;
- вспышка или резкое движение тоже добавляют риск.
- жест помощи сразу сильно повышает риск и запускает тревогу.

Риск постепенно уменьшается, если новых опасных событий нет. Это нужно, чтобы одиночный случайный шум не держал тревогу постоянно.

### `EvidenceRecorder`

Сохраняет доказательства события.

Когда тревога активируется, программа сохраняет:
- изображение с камеры `.jpg`;
- описание события `.json`.

Файлы появляются в папке:

```text
emergency_evidence
```

### `EmergencyAction`

Отвечает за финальное действие после тревоги.

В DEMO-режиме он только пишет, что действие было бы выполнено.

В REAL-режиме может:
- сделать звонок через Twilio;
- выполнить локальную команду из `EMERGENCY_ACTION_COMMAND`.

Важно: реальный режим нужно включать вручную:

```powershell
$env:EMERGENCY_REAL_ACTION="1"
```

### `ShelterManager`

Отвечает за вкладку укрытия в формате ленты.

Он читает файл:

```text
shelters.json
```

Чтобы программа могла найти ближайшее укрытие, нужно добавить реальные укрытия в `shelters.json` и задать координаты пользователя:

```powershell
$env:USER_LAT="50.4501"
$env:USER_LON="30.5234"
```

Также добавлена интеграция с Google Maps Places API. Если указать ключ Google Maps, программа сможет искать укрытия и станции метро рядом с координатами пользователя:

```powershell
$env:GOOGLE_MAPS_API_KEY="your_google_maps_key"
$env:USER_LAT="50.4501"
$env:USER_LON="30.5234"
```

Дополнительные настройки:

```powershell
$env:GOOGLE_MAPS_SHELTER_QUERY="бомбоубежище укриття shelter метро станция метро subway metro station"
$env:GOOGLE_MAPS_RADIUS_M="5000"
```

Если API-ключа нет, клавиша `M` все равно откроет обычный поиск/маршрут в Google Maps через браузер.

Клавиши:
- `1` - вкладка мониторинга;
- `2` - лента укрытий;
- `G` - обновить поиск укрытий через Google Maps Places API;
- `L` - регистрация/вход через Google;
- `M` - открыть карту/поиск укрытия.

При тревоге программа сама переключается на вкладку-ленту укрытий. Если `GOOGLE_MAPS_API_KEY`, `USER_LAT` и `USER_LON` заданы, поиск через Google Maps запускается автоматически. Метро учитывается как возможное укрытие, поэтому станции метро тоже могут появляться в ленте. Первый элемент ленты - ближайший найденный вариант.

### `AlarmController`

Управляет тревогой.

Он делает:
- включает звуковой сигнал;
- запускает обратный отсчет;
- разрешает отменить тревогу;
- разрешает подтвердить действие сразу;
- запускает `EmergencyAction`, если время вышло.

Клавиши:
- `Q` - выйти;
- `X` - отменить тревогу;
- `C` - подтвердить действие.
- `1` - мониторинг;
- `2` - лента укрытий;
- `G` - поиск Google Maps;
- `L` - регистрация/вход через Google;
- `M` - открыть карту.

### `draw_overlay`

Рисует текст поверх изображения с камеры.

На экране показывается:
- текущий риск;
- DEMO или REAL режим;
- причина тревоги;
- параметры движения и вспышки;
- подсказки по клавишам.

### `open_camera`

Открывает камеру и задает качество изображения.

Если камера не открылась, программа останавливается с ошибкой.

### `main`

Главная функция программы.

Здесь все блоки собираются вместе:
- создаются настройки;
- запускается микрофон;
- открывается камера;
- начинается бесконечный цикл обработки;
- анализируется звук, видео и жесты;
- отображается привязанный Google-аккаунт;
- обновляется риск;
- включается тревога;
- при тревоге открывается лента укрытий;
- обрабатываются клавиши;
- при выходе освобождаются камера и микрофон.

## Как запустить

Установить зависимости:

```powershell
pip install -r requirements.txt
```

Запустить:

```powershell
python ".\rescue app.py"
```

## Мобильное расширение на KivyMD

Для мобильной версии добавлены файлы:

```text
mobile_rescue_app.py
main.py
buildozer.spec
mobile_requirements.txt
```

`mobile_rescue_app.py` - отдельный KivyMD-интерфейс для Android/мобильного формата. Он не ломает десктопный `rescue app.py`.

Что уже есть в мобильной версии:
- главный экран SOS;
- кнопка набора номера `0977477926`;
- лента ближайших укрытий и станций метро;
- поиск через Google Maps Places API;
- открытие маршрута в Google Maps;
- чтение локального `shelters.json`;
- отображение Google-профиля из `google_user_profile.json`, если он уже создан;
- запрос Android permissions для геолокации, камеры, микрофона и интернета.

Локальный запуск UI:

```powershell
pip install -r mobile_requirements.txt
python ".\mobile_rescue_app.py"
```

Сборка APK обычно делается через Linux или WSL, потому что Buildozer ориентирован на Android toolchain:

```bash
buildozer android debug
```

`main.py` нужен как точка входа для Buildozer.

Важно: мобильная версия сейчас содержит интерфейс, SOS, карту, геолокацию и ленту укрытий. Полный мобильный анализ камеры/микрофона лучше подключать следующим этапом после проверки Android permissions и фоновых ограничений устройства.

## Как включить реальный звонок через Twilio

Пример:

```powershell
$env:EMERGENCY_REAL_ACTION="1"
$env:EMERGENCY_NUMBER="0977477926"
$env:GOOGLE_OAUTH_CLIENT_ID="your_client_id"
$env:GOOGLE_OAUTH_CLIENT_SECRET="your_client_secret"
$env:TWILIO_ACCOUNT_SID="..."
$env:TWILIO_AUTH_TOKEN="..."
$env:TWILIO_FROM_NUMBER="..."
python ".\rescue app.py"
```

Лучше сначала тестировать в DEMO-режиме или на личный номер, чтобы не создать ложный экстренный вызов.

## Что можно улучшить дальше

Сейчас это прототип с простыми признаками звука и видео. Для более настоящего ИИ можно добавить:
- модель распознавания звуков, например gunshot/scream/siren classifier;
- распознавание человека, падения, оружия или дыма на видео;
- отправку геолокации;
- отправку SMS родственнику;
- запись короткого видео до и после тревоги;
- мобильное приложение на Android;
- кнопку "я в безопасности";
- список доверенных контактов перед звонком в экстренную службу.
